## Object Types
Un Objeto es una **colección de datos con ciertos comportamientos asociados**. Pero, ¿cómo distinguimos tipos de objetos? Las manzanas y las naranjas son ambos objetos, pero no se pueden comparar directamente. E.g la siguiente regla: 

 > Las manzanas van en barriles, las naranjas van en cestas

En esta situación tenemos 4 tipos de objetos **manzanas**, **barriles**, **naranjas** y **cestas**.  

## Classes
En el modelo Object-Oriented ([[Introducing Object-Oriented]]) los tipos de objetos son considerados **clases**. 
- **Clase**: Sería como el **plano/molde, el concepto/idea** de lo que iría adentro. E.g: la clase "molde de galletas".
- **Objeto**: Sería la **cosa real creada, algo concreto y real** con sus propios datos o valores (distintos objetos). E.g: galleta 1 (la de Tobi), galleta 2 (la de Gonza), galleta 3 (la de Ana). 

~~~txt
┌─────────────────────┐        ┌──────┐ ┌──────┐ ┌──────┐
│  Molde de galleta   │  →→→   │  🍪  │ │  🍪  │ │  🍪  │
│  (la forma/plano)   │        │ #001 │ │ #002 │ │ #003 │
└─────────────────────┘        └──────┘ └──────┘ └──────┘
       CLASE                          OBJETOS
~~~

- El **molde** define la forma, el tamaño, el diseño → eso es la **Clase**
- Cada **galleta individual** producida con ese molde → eso es un **Objeto**
El molde existe **una sola vez**. Las galletas pueden existir **miles de veces**, cada una siendo una cosa concreta e individual.
